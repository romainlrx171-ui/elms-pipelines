Comment ça marche

3 programmes isolés
Chaque prospect est verrouillé à un seul programme à la création. SQLite refuse tout changement de programme via trigger d'immutabilité.
Garde-fous SMTP
Avant chaque envoi : (1) prospect.program == campaign.program, (2) email pas actif dans 2 programs simultanément, (3) spacing 48h+ sur même domaine entre programs.
Footer par programme
Chaque mail signe avec un footer différent selon le programme (ELMS · Spa · Stratos). Visible dans l'audit trail.
Dashboard anonymisé
Aucun email/téléphone/LinkedIn n'est exporté vers GitHub. Tu vois has_email: ✓ mais l'adresse ne quitte jamais la DB locale.
